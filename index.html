JavaScript

Características del lenguaje

  • Tipado dinámico y débil: los tipos se verifican en tiempo de ejecución y pueden cambiar.
  • Paradigma orientado a objetos con herencia por prototipos (no de clases reales).
  • Sintaxis similar a Java; tiene características de Lisp (funciones de primer orden).
  • Estándar mundial: ECMAScript.
  • Ambientes: Browser (accede a document, DOM) y Node.js (no tiene document).

Principales tipos de datos

TipoNotas clave
Números64 bits, punto flotante. Incluye NaN e Infinity.
StringsComilla simple o doble. + concatena. Índice empieza en 0. Métodos: indexOf, substring, replace, trim, split.
Booleanostrue / false
ObjetosColecciones de propiedades (datos y métodos). Se crean con literales {}.
ArreglosColecciones por índice. var a = [5,3,7] o new Array(). Propiedad length.

Variables y ámbitos

  • var dentro de función → ámbito local a la función.
  • var fuera de función → global (objeto global / window).
  • let / const → ámbito de bloque ({}).
  • Variables no inicializadas = undefined (distinto de null).
  • Funciones sin return devuelven undefined, excepto si se llaman con new.

Funciones – múltiples sintaxis

// Declaración clásica
function suma(x, y) { return x + y; }

// Expresión (asignada a variable)
var suma = function(x, y) { return x + y; };

// Arrow function
var suma = (x, y) => x + y;

// Auto-invocada (IIFE)
((x, y) => { return x + y; })(1, 2);

Las funciones son elementos de primer orden: pueden asignarse a variables y pasarse como parámetros.

Arreglos – métodos importantes

  • push(e) – agrega al final; pop() – elimina del final.
  • shift() – elimina del inicio; unshift(e) – agrega al inicio.
  • splice(i, c, e) – en posición i elimina c elementos e inserta e.
  • slice(), concat(), join(), sort().
  • forEach(), filter(), find(), findIndex(), map().

Recorridos: for, for (i in a), a.forEach(fn).

Objetos y constructores

// Literal de objeto
var p = { cedula: '001', nombre: 'Juan' };
p.edad = 30; // agregar propiedad dinámicamente

// Función constructora
function Persona(c, n) { this.cedula = c; this.nombre = n; }
var p = new Persona('001', 'Juan');
// new: crea objeto vacío, asigna this, ejecuta función, retorna objeto

Herencia por prototipos

  • Cada objeto tiene una referencia __proto__ a su prototipo.
  • Al buscar una propiedad: si no existe, se navega la cadena de prototipos hasta encontrarla o llegar a un objeto sin prototipo.
  • Las funciones tienen la propiedad prototype. Al llamar con new, el objeto creado hereda de ese prototype.

Clases en ES6 (azúcar sintáctico)

class Animal {
  constructor(nombre) { this.nombre = nombre; }
  hablar() { return this.nombre + ' hace un sonido'; }
}

class Perro extends Animal {
  constructor(nombre) { super(nombre); }
  hablar() { return super.hablar() + ' (guau)'; }
}
let d = new Perro('Rex');
d.hablar(); // "Rex hace un sonido (guau)"

⚠️ Las clases son azúcar sintáctico sobre el patrón de función constructora. JS sigue usando herencia por prototipos internamente.

JSON

  • Sublenguaje de JS para representar datos como texto. Soporta objetos, arreglos y primitivos.
  • JSON.stringify(obj) → serializa objeto a texto JSON.
  • JSON.parse(texto) → deserializa texto JSON a objeto JS.
let p = {cedula:'001', nombre:'Juan', edad:30};
let texto = JSON.stringify(p);
// → '{"cedula":"001","nombre":"Juan","edad":30}'
let obj = JSON.parse(texto); // → objeto JS de vuelta

Operador ?. y ??

  • x.y?.z: si x.y existe retorna x.y.z, sino undefined (sin lanzar error).
  • Equivale a x.y && x.y.z.
  • valor ?? 'default': si valor es null o undefined, usa 'default'.

DOM – Document Object Model

¿Qué es el DOM?

  • Especificación del W3C para representar documentos HTML/XML como árbol en programas (JS, Java...).
  • Cuando el browser carga una página, crea un árbol de nodos y un ámbito de ejecución de JS.
  • Cada etiqueta HTML → un nodo; sub-etiquetas → nodos hijos.

Jerarquía de interfaces (de más general a más específico)

EventTarget
  └── Node
        ├── Text
        └── Element
              └── HTMLElement
                    ├── HTMLInputElement
                    ├── HTMLFormElement
                    └── HTMLBodyElement ...
Document (también hereda de Node → EventTarget)

⚠️ Todos los nodos pueden recibir eventos porque Node hereda de EventTarget.

El objeto window

  • Es el ambiente global de JS en el browser.
  • Variables/funciones declaradas en el top-level quedan como propiedades de window.
  • Contiene: document, history, location, navigator, JSON, XMLHttpRequest, etc.
  • No existe en Node.js.

  • EventTarget: interfaz que implementan los nodos que reciben eventos.

Fetch API

¿Por qué Fetch?

  • El antiguo XMLHttpRequest (XHR): complejo, callbacks, inconsistencias entre browsers.
  • Fetch API: forma moderna de peticiones HTTP asíncronas en JS.
  • Usa Promises en lugar de callbacks.
  • Soportado por todos los browsers desde 2016/2017 (excepto IE).
  • Por seguridad, solo carga datos del mismo servidor salvo que se habilite CORS.

Uso básico

// Con .then() encadenado
fetch('https://api.example.com/datos.json')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(err => console.log(err));

// Con async/await (más legible)
async function cargarDatos() {
  const response = await fetch('https://api.example.com/datos.json');
  const data = await response.json();
  console.log(data);
}
cargarDatos();

Objeto Response – propiedades clave

PropiedadDescripción
response.okBoolean: true si HTTP 200-299
response.statusCódigo HTTP (200, 404, 500...)
response.statusText"OK", "Not Found", etc.
response.urlURL final (tras redirects)
response.type"basic" o "cors"

Métodos para leer el body (solo se puede leer UNA vez)

  • response.json() → Promise → objeto JS
  • response.text() → Promise → string
  • response.formData() → Promise → FormData
  • response.blob() → Promise → datos binarios con tipo
  • response.arrayBuffer() → Promise → representación binaria

Manejo de errores

⚠️ Fetch NO rechaza la Promise por errores HTTP (404, 500). Solo rechaza por errores de red. Siempre verificar response.ok.

fetch(url)
  .then(response => {
    if (!response.ok) {
      throw Error(response.statusText); // manejar 4xx / 5xx
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(err => console.log(err));

Opciones del fetch (objeto init)

fetch(url, {
  method: 'POST',              // GET, POST, PUT, DELETE
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(objeto), // serializar a JSON para enviar
  mode: 'cors',                // cors | no-cors | same-origin
  credentials: 'include'       // omit | same-origin | include (para cookies)
})

CRUD con Fetch (patrón del proyecto)

// GET – listar todos
const req = new Request(backend + '/personas', {method:'GET', headers:{}});
const response = await fetch(req);
if (!response.ok) { alert("Error: " + response.status); return; }
const personas = await response.json();

// POST – crear nuevo
let req = new Request(backend + '/personas', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify(persona)
});
const response = await fetch(req);

// GET por ID – obtener uno para editar
const req = new Request(backend + '/personas/' + persona._id, {method:'GET'});
const response = await fetch(req);

Fetches en paralelo con Promise.all()

const urls = [url1, url2, url3];
const promises = urls.map(url => fetch(url).then(r => r.json()));

Promise.all(promises)
  .then(results => results.forEach(data => console.log(data)))
  .catch(e => console.error(e));

Cancelar petición con AbortController

const controller = new AbortController();

fetch(url, { signal: controller.signal })
  .then(response => response.json())
  .catch(err => console.log(err.message));

// Para cancelar en cualquier momento:
controller.abort();

REST y RESTful Services

¿Qué es REST?

REST: REpresentational State Transfer – Transferencia de una representación del estado de un recurso.

  • Recurso: objeto (OOP), registro de DB, imagen, etc.
  • Representación: JSON, XML, etc.
// Objeto Persona representado en JSON:
{ "Id": "001", "Nombre": "Juan" }

// Misma representación en XML:
<Persona><Id>001</Id><Nombre>Juan</Nombre></Persona>

Métodos HTTP y operaciones CRUD

Método HTTPOperaciónEjemplo URL
GETLeer / listar/personas o /personas/1
POSTCrear/personas
PUTActualizar (reemplaza completo)/personas/1
DELETEEliminar/personas/1

Arquitectura de una app web moderna

  • Views/Presentation: páginas HTML recibidas por el browser.
  • API RESTful: retorna datos JSON consumidos por el JS del cliente.

Flujo: Browser → petición HTTP → Servidor Spring/Tomcat → procesa → retorna JSON → JS actualiza el DOM.

Implementación con Spring (servidor)

El procesador RESTful de Spring debe:

  1. Identificar servicio y método según el URI y el método HTTP.
  2. Convertir el body de la petición de JSON a objeto Java (según Content-Type).
  3. Invocar el método del API con ese objeto.
  4. Recibir el resultado Java y convertirlo a JSON (según Accept).
  5. Devolver la respuesta con el HTTP status correspondiente.
  6. Si hay excepción → mapearla a un HTTP status de error (ej. 404) y devolver ese status.

Códigos de respuesta HTTP

CódigoSignificadoCuándo usarlo
200 OKÉxitoGET, PUT exitosos
201 CreatedRecurso creadoPOST exitoso
204 No ContentÉxito sin bodyDELETE exitoso
400 Bad RequestPetición malformadaDatos inválidos
404 Not FoundNo encontradoRecurso inexistente
500 Internal Server ErrorError en servidorExcepción no manejada

Headers HTTP importantes

  • Content-Type: application/json → el body que ENVÍO es JSON.
  • Accept: application/json → quiero RECIBIR JSON en la respuesta.
  • CORS (Cross-Origin Resource Sharing): mecanismo para permitir peticiones desde un origen diferente al del servidor.

React

¿Por qué React?

  • Creado por Facebook. Licencia MIT. Usado en +380,000 sitios.
  • Problemas del DOM vanilla/jQuery: sin estructura, difícil para SPAs, eventos complejos.
  • React define todo como componentes, maneja actualizaciones automáticamente.
  • Agrega JSX: sintaxis HTML dentro de JS. JSX compila a React.createElement().

JSX y tipos de componentes

// Componente de clase (puede tener state)
class MiComponente extends React.Component {
  render() { return <div>Hola Mundo</div>; }
}

// Componente funcional (moderno, con Hooks)
function MiComponente() {
  return <div>Hola Mundo</div>;
}

// Arrow function (equivalente)
const MiComponente = () => <div>Hola Mundo</div>;

// JSX compila a:
React.createElement('div', null, 'Hola Mundo');

State y Props

StateProps
AccesoLectura y escrituraSolo lectura
Pertenece aEl componente mismoViene del padre
ActualizarsetState() o setter de useStateNo se modifica desde el hijo

// Clase

// Hook funcional

Eventos en React

Nombres en camelCase. Se asignan como propiedades JSX (no strings).

<button onClick={this.handleClick}>Clic</button>
<button onClick={() => this.handleClick(param)}>Clic con param</button>
<input onChange={(e) => setValor(e.target.value)} />
// Otros: onSubmit, onDoubleClick, onContextMenu, onChange, onKeyDown...

Formularios: Controlled vs Uncontrolled

Componente Controlado: React gestiona el valor via state. React es la fuente de verdad.

const [nombre, setNombre] = useState('');
<input value={nombre} onChange={(e) => setNombre(e.target.value)} />
  • ✅ Validación en tiempo real, control total, debug fácil, formularios complejos.
  • ❌ Más código, más re-renders por cada tecla.

Componente No Controlado: el DOM gestiona el valor. React lo lee con ref cuando lo necesita.

const inputRef = useRef(null);
<input ref={inputRef} />
// Al hacer submit:
const valor = inputRef.current.value;
  • ✅ Menos código, mejor rendimiento, familiar para devs de HTML.
  • ❌ Sin acceso en tiempo real, validación más difícil.

Context API – evitar prop drilling

Prop drilling: pasar props por componentes intermedios que no los necesitan.

// 1. Crear contexto
import { createContext, useState, useContext } from 'react';
const CounterContext = createContext({ count: 0, setCount: () => {} });

// 2. Crear Provider
const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);
  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

// 3. Consumir con useContext (en cualquier componente descendiente)
const { count, setCount } = useContext(CounterContext);

// 4. Envolver la app con el Provider
<CounterProvider>
  <App />
</CounterProvider>

Usos comunes: autenticación, tema (dark/light), carrito, idioma.

React Router v7 (modo declarativo)

import { BrowserRouter, Routes, Route } from 'react-router';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="*" element={<NoMatch />} />  {/* 404 */}
      </Routes>
    </BrowserRouter>
  );
}
ComponenteRol
<BrowserRouter>Envuelve toda la app. Usa HTML5 History API.
<Routes>Reemplaza <Switch> de v5. Soporta rutas relativas y anidadas.
<Route path element>Define una ruta y el componente a renderizar.

Tres modos: declarativo (básico), data (fetching integrado), framework (como Next.js). Instalación: npm install react-router
